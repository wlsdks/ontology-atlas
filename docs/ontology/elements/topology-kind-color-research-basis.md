The topology kind-color contract is grounded in two public design/accessibility references:

- W3C WCAG 2.2 Understanding SC 1.4.1: color can support meaning, but it should not be the only visual way to distinguish information.
- Okabe-Ito / Color Universal Design style palettes: use hue-separated colors that remain distinguishable for common color-vision differences.

Ontology Atlas applies those references pragmatically: domain, capability, element, and unknown get distinct hues, while legend labels and node-size hierarchy remain available as non-color cues.
